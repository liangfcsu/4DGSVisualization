# 训练实时可视化 - 修复说明

## ✅ 已修复的问题

### 1. 右侧面板训练信息显示

**问题：** 训练信息卡片不显示

**原因：** Qt的 `setVisible()` 方法在某些情况下不能正确工作

**解决方案：** 
- 不再隐藏/显示卡片，让它始终可见
- 未训练时显示默认值 "—"
- 训练时显示实时数据

**效果：**
```
TRAINING INFO
状态      训练中
迭代      5,000 / 30,000 (16%)
Loss      0.009876
点数      75,000
```

### 2. 播放窗口实时显示训练结果

**问题：** 训练过程中播放窗口不更新

**原因：** 
- 只在最终迭代保存checkpoint
- 检测新PLY文件的频率不够

**解决方案：**
- 增加checkpoint保存点：500, 1000, 2000, 3000, 5000, 7000, 10000, 15000, 20000, 最终
- 检测到保存信号后等待0.5秒让文件写入完成
- 添加详细的调试信息追踪整个流程

**实时加载流程：**
```
训练到500次迭代
   ↓
保存checkpoint: [ITER 500] Saving Gaussians
   ↓
等待0.5秒（文件写入）
   ↓
检测新的PLY文件: point_cloud/iteration_500/point_cloud.ply
   ↓
加载到渲染器
   ↓
播放窗口显示最新点云！
```

## 🎯 使用方法

### 1. 启动播放器
```bash
python player.py data/New_Folder --sparse data/fram8/0
```

### 2. 开始训练

1. **打开左侧面板** → 找到 "训练控制" 部分
2. **选择训练数据** → 点击 "选择训练数据" 按钮
   - 选择包含 `images/` 和 `sparse/` 的目录
3. **选择输出目录** → 点击 "选择输出目录" 按钮
   - 选择训练结果保存位置
4. **设置迭代次数** → 默认30000（可调整）
5. **点击开始训练** → "▶ 开始训练" 按钮

## 🔍 故障排查

### 如果右侧面板不显示训练信息

1. 检查控制台是否有 `[DEBUG] 首次显示训练信息卡片`
2. 确认左侧面板显示了训练信息
3. 如果左侧有、右侧没有，说明 `_update_training_status()` 没有调用右侧面板的更新

### 如果播放窗口不更新

1. **检查是否开始训练**
   - 左侧面板显示 "状态: 训练中"
   - 控制台输出训练命令

2. **检查checkpoint是否保存**
   - 控制台查找：`[ITER 500] Saving Gaussians`
   - 检查输出目录：`output/point_cloud/iteration_500/`

3. **检查PLY文件检测**
   - 控制台查找：`[检查PLY] 检测到新的checkpoint`
   - 如果没有，检查 `point_cloud` 目录是否存在

4. **检查PLY加载**
   - 控制台查找：`[可视化] 点云加载成功`
   - 如果加载失败，查看错误信息

### 常见问题

**Q: 训练开始了但窗口没有变化？**
A: 等待第一个checkpoint保存（迭代500），控制台会输出 `[ITER 500] Saving Gaussians`

**Q: 控制台没有 `[检查PLY]` 信息？**
A: 检查训练命令是否正确启动，查看是否有错误信息

**Q: PLY文件存在但没有加载？**
A: 检查文件路径和权限，确认 `GaussianPointCloud` 类可以读取文件

## ⚡ 性能说明

- **状态更新频率**：每0.5秒更新一次UI（避免刷新过频繁）
- **PLY检测频率**：每20行输出检查一次
- **Checkpoint保存**：每次保存需要几秒到十几秒（取决于点云大小）
- **PLY加载**：取决于点云大小，通常1-5秒
