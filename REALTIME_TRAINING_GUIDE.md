# 实时训练过程可视化 - 使用说明

## 🎉 功能概述

你现在可以在4DGS播放器中**实时观看**3D Gaussian Splatting的训练过程！

### 核心特性

✅ **后台训练** - 训练在独立线程中运行，不阻塞UI  
✅ **实时更新** - 每100次迭代自动导出并加载高斯点云到渲染器  
✅ **动态显示** - 看着点云从稀疏到密集，从模糊到清晰  
✅ **训练监控** - 实时显示迭代次数、Loss、点数等指标  
✅ **随时停止** - 可以随时中断训练并保留当前进度  

## 🚀 快速开始

### 1. 启动播放器

```bash
conda activate 3dgs
python player.py
```

### 2. 在UI中配置训练

1. 打开左侧面板
2. 展开 **"训练控制"** 部分
3. **选择训练数据** - 点击按钮，选择包含 `images/` 和 `sparse/` 的文件夹
4. **选择输出目录** - 点击按钮，选择训练结果保存位置
5. **设置迭代次数** - 默认30000（可根据需要调整）

### 3. 开始训练并观看

1. 点击 **"▶ 开始训练"** 按钮
2. 播放器会显示提示："训练已启动，正在实时显示训练过程..."
3. **窗口标题**会显示当前迭代次数： `[训练迭代: 100]`
4. **场景视图**每100次迭代自动更新一次

### 4. 观察训练过程

你会看到：
- **初始阶段** (0-500次): 点云稀疏，随机分布
- **密集化阶段** (500-15000次): 点云逐渐增多，形状逐渐清晰
- **优化阶段** (15000-30000次): 点云数量趋于稳定，细节不断优化

```
├── ───────────────
├── ▶ 开始训练 [按钮]
└── ⏹ 停止训练 [按钮]
```

### 实时指标含义

- **状态**: 当前训练状态（未开始/训练中/已停止/已完成/错误）
- **进度**: 当前迭代次数 / 总迭代次数
- **Loss**: 当前损失值（越小越好）
- **点数**: 当前高斯点的数量（会动态变化）

- 训练结果已保存到输出目录

## ⚙️ 高级配置

### 调整可视化更新频率

在 `training_manager.py` 中修改：

```python
self.visualization_update_interval = 100  # 改为 50 或 200
```

- **更小的值** (如50): 更频繁更新，更流畅，但可能影响训练速度
- **更大的值** (如200): 更新间隔更长，训练速度更快

### 修改显示提示频率

在 `player.py` 的 `_load_training_ply` 方法中：

```python
if iteration % 500 == 0:  # 改为 100 或 1000
    self.toast.show_message(...)
```

## 💡 提示和技巧

### 性能优化

1. **降低可视化频率**: 如果训练速度慢，增大 `visualization_update_interval`
2. **关闭其他程序**: 确保GPU专注于训练
3. **调整图像分辨率**: 使用 `--resolution` 参数降低训练图像分辨率

### 观察要点

训练过程中关注：
- **点数变化**: 应该逐渐增加到稳定
- **Loss下降**: 应该持续下降
- **视觉质量**: 细节应该越来越清晰

### 常见问题

**Q: 为什么点云更新很慢？**  
A: 正常现象，训练需要时间。可以降低可视化频率提升训练速度。

**Q: 训练中途可以暂停吗？**  
A: 当前版本只能停止。未来版本会支持暂停/恢复。

**Q: 可以同时训练多个场景吗？**  
A: 当前版本一次只能训练一个场景。

**Q: 训练失败怎么办？**  
A: 检查训练数据是否正确、CUDA是否可用、查看错误提示。

## 🔧 故障排除

### 训练无法启动

```bash
# 检查CUDA
python -c "import torch; print(torch.cuda.is_available())"

# 检查数据路径
ls 训练数据路径/images/
ls 训练数据路径/sparse/0/
```

### 可视化不更新

- 检查console输出是否有 "可视化PLY已导出" 消息
- 确认 `_training_visualization_enabled = True`
- 查看临时目录是否有PLY文件生成

### 内存不足

- 降低迭代次数先测试
- 使用 `--resolution` 限制图像分辨率
- 减少训练图像数量
